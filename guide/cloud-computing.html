Onboarding Guide: Cloud Computing Data

Getting Started

You've been assigned as the data holder for Cloud Computing, which covers the impact of the company's cloud infrastructure from services like AWS, Azure, or GCP.

Due to complications in accessing this data, the method for providing the correct information varies by provider.

For this section, we are referring to Infrastructure-as-a-Service (IaaS) rather than consumer-facing services like websites, SharePoint, or Google Drive. The impact of smaller-scale items like these is negligible and considered immaterial to the overall cloud impact.

This guide will help you format the data correctly to build this part of the company's impact assessment.

You'll see the following task in your to-do list here: vso.alectro.io/todo-list

Finding and Requesting Data

Data can be downloaded directly from your cloud provider:

  • For Azure and GCP, this information can be accessed directly through the tools they provide for reporting GHG Protocol Scope 1-3 data.
  • For other providers, including AWS, a raw data approach is required.

The most critical factors in determining impact are monthly uptime and the geographical region of the cloud servers.

We've listed how to export this data here:

Amazon Web Services

You can access the from your AWS services by following the steps below:

  1. Go to AWS Cost Management > Cost Explorer > New cost and usage report
  2. Apply the following filters:
    1. For the date range, choose the Reporting Period Year (or as much of it as you are able to select if you cannot go back all the way)
    2. Granularity: Set to Monthly
    3. Dimension: Choose Region
  3. Then for each of the following combinations, you can download a CSV of the data:
    1. For Compute, you need to filter the Usage type group to EC2: Running Hours
    2. For Storage, you need to filter to S3: Storage - Standard
    3. For RDS, you need to filter to RDS: Running hours
    4. For Networking, you need to filter to give each type of Data Transfer - xxx (In) / Data Transfer - xxx (Out). You can add all of these in one go.

    4. You should see two graphs (as shown in the screenshot)â€”one displaying cost and the other showing usage.

    Click Download CSV to download the data, and upload the data, or send it over to us!
    This will give you a download of each months cloud computing use, split by region, for the period of analysis.

Google Cloud Platform

The Carbon Footprint dashboard displays estimated greenhouse gas emissions associated with the usage of covered Google Cloud services for the selected billing account. You can follow instructions to view and export the data from the dashboard.

You should export the data from the Market-based emissions tab: This tab displays emissions data broken down by scope 1, scope 2 market-based, and scope 3 GHG emissions.

Microsoft Azure

You can export your emissions data in CSV format from the Emissions Trends page and the Emissions Details page in Azure carbon optimization in the Azure portal.

You can follow the instructions here: Export emissions data to a CSV file using the Azure portal - this will give us detailed information so we can get more accurate data on a monthly basis.

This will help you to download the Emissions Trends data that includes the last 12 month's data for the selected filters, including:

  • Scope 1, 2, 3 and total emissions
  • Carbon intensity

Download the data, and upload or send it over to us!
